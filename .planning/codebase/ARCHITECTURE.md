## Data Flow

### Pricing Pipeline (Core Business Flow)
```
Carrier Rate Files (PDF/Excel)
    → Pricing_Engine/rate_importer.py (OCR + parse)
    → Pricing_Engine/clean_parquet.py (normalize)
    → Parquet dataset (single source of truth)
    → ERP/core/refresh.py (Parquet → ERP_Master.xlsm)
    → ERP_Master.xlsm (operational use)
```

### Query Flow (Bot/API/WebApp)
```
User request
    → Bot: query_engine.py → Parquet (pandas/pyarrow)
    → API: data_access.py → DuckDB → Parquet
    → WebApp: fetch('/api/...') → FastAPI → data_access.py
```

### Intelligence Flow
```
Parquet data
    → intelligence/anomaly_detector.py (Drewry 15%/30%)
    → intelligence/rate_predictor.py
    → intelligence/alert_dispatcher.py → Telegram notification
    → ERP/intelligence/price_alerts.py → ERP dashboard
```

## Entry Points

| WebApp | `webapp/` | `npm run dev` (port 3002/3003) |

## Key Abstractions
- **DAL (Data Access Layer):** `api/data_access.py` — all API queries go through this
- **Query Engine:** `TelegramBot/query_engine.py` — bot-specific Parquet queries
- **Markup Engine:** `TelegramBot/markup_engine.py` — pricing formula implementation
- **Config:** `TelegramBot/config.py` — centralized path resolution for bot
- **Event Bus:** `api/event_bus.py` — internal async event routing
