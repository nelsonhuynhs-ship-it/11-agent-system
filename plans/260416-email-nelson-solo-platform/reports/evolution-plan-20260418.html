Alternative 50-line cho Panjiva cleaning

import pandas as pd, janitor  # pyjanitor
from rapidfuzz import fuzz, process

df = pd.read_excel("panjiva_export.xlsx")
df = (df.clean_names()                          # snake_case columns
        .remove_empty()
        .dropna(subset=["consignee"])
        .drop_duplicates(subset=["consignee", "email"]))

master = pd.read_excel("cnee_master_v2_final.xlsx")
def match_existing(name):
    m, score, _ = process.extractOne(name, master["COMPANY"], scorer=fuzz.WRatio)
    return m if score > 85 else None
df["matched_master"] = df["consignee"].apply(match_existing)
# Rows with NULL matched_master = new prospects → append with tier=NEW
df.to_csv("cleaned.csv", index=False)

Local LLM vs API

Ollama local llama3.1:8b được chọn — batch 100 email trong 2-3 phút, offline, PII không rời máy. Claude API nhanh hơn 10× ($0.10/1K email) nhưng không đáng đổi privacy.

Pre-built OSS để fork

  • lucasastorian/llmwiki — gần nhất, built cho research docs. Fork effort ~2-3 ngày cho MVP email parser + CNEE schema
  • LLM Wiki v2 gist — extension của Karpathy pattern

Privacy (solo operator standard 2026)

  • Local disk + BitLocker (Windows) = đủ cho solo B2B NVOCC
  • VPS vault: encrypt với age CLI
  • Không cần enterprise tooling — không có team, không có GDPR audit
