🛡 Shipper Blacklist — Không đụng team VN

Shipper VN trùng nhiều với team nội bộ đang làm. Cần cơ chế chặn để không chào giá cho khách team đã chăm sóc.

File blacklist

D:/OneDrive/NelsonData/email/vn_team_overlap.xlsx — 3 sheet:
Sheet 1: BY_COMPANY | COMPANY_NAME_EXACT | COMPANY_NAME_FUZZY | TEAM_MEMBER | SINCE_DATE | NOTE | (ví dụ: "ABC Wood Co Ltd" — Johnny đang chăm từ 2025-10) Sheet 2: BY_EMAIL | EMAIL_OR_DOMAIN | TEAM_MEMBER | NOTE | (ví dụ: "@abcwood.com" toàn domain bị block) Sheet 3: BY_PHONE | PHONE_E164 | TEAM_MEMBER | NOTE |

Match logic (3 lớp)

Trước mỗi send tới SHIPPER row: ├─ Lớp 1 EXACT company_name.lower().strip() == blacklist[COMPANY_NAME_EXACT] │ → nếu match → BLOCK + flag VN_TEAM_OVERLAP_FLAG=Y │ ├─ Lớp 2 FUZZY fuzzy_ratio(company, blacklist[COMPANY_NAME_FUZZY]) > 85 │ → nếu match → HOLD for review (không block hẳn, chờ Nelson confirm) │ └─ Lớp 3 EMAIL/PHONE domain match hoặc phone E.164 match → BLOCK luôn

🛡 Safety — Không làm hư data priority

Rule sắt đá — 5 cột không bao giờ overwrite:
  1. EMAIL_STATUS · WA_STATUS · LI_STATUS
  2. SEND_COUNT_* (email/wa/li)
  3. LAST_SENT_*
  4. REPLY_STATUS
  5. TIER nếu = CUSTOMER hoặc VIP → khóa vĩnh viễn

3 lớp bảo vệ

  1. Backup trước thao tác — rotation 14 bản, path backups/contact_unified_v6.backup_YYYYMMDD_HHMM.xlsx
  2. Dry-run mode — mọi script có --dry-run in diff preview
  3. Audit log CSV — từng row: action (NEW/UPDATE/SKIP), cột nào đổi, value trước/sau

Customer protection flow (trước mọi bulk send)

Check: ├─ TIER == CUSTOMER → SKIP bulk (manual 1-1 only) ├─ SIDE == SHIPPER và chưa Activate → SKIP ├─ VN_TEAM_OVERLAP_FLAG == Y → SKIP ├─ REPLY_STATUS != null → SKIP (đã có relationship) ├─ Cooldown 14d → SKIP ├─ TYPO_SUSPECT → HOLD Nelson review └─ Channel-specific: ├─ EMAIL: EMAIL_STATUS not VALID → SKIP ├─ WA: WA_STATUS != EXISTS → SKIP └─ LI: LI_STATUS != CONNECTED → SKIP (connect first)
Rollback (1-click từ Contacts tab):
  • Nút ⏪ Rollback last migration restore backup mới nhất
  • Git tag mỗi schema change
  • Audit log là evidence để debug mọi thay đổi
