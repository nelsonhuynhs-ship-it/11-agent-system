⚠️ Reality Check — Excel native có đủ value không?

Anh đã chốt Combo A. Trước khi build, em phải honest về Excel limitations.

🤔 Anh đúng skeptical: v1/v2/v3 đều fail. Em phải verify v4 có thực sự đẹp + work trước khi build.

🔍 3 Approaches build Combo A — em so sánh thật

A. Pure Excel VBA
(plan ban đầu)

  • VBA code trong Excel
  • UserForm popup — visual ugly Win95
  • Chart trong UserForm phức tạp, render xấu
  • Latency 5-30s per click (query 6.6M parquet rows từ VBA)
  • VBA không có pandas/duckdb → phải Shell call Python
  • Maintenance nightmare khi schema change
❌ KHÔNG nên — anh sẽ thất vọng v4

B. Excel + Helper sheets
(half-working)

  • Pre-compute data vào sheet ẩn (refresh-v14.py update)
  • VBA query helper sheet (fast)
  • UserForm popup vẫn ugly
  • Charts limited to Excel native
  • Data stale nếu refresh không trigger
⚠️ OK nhưng visual vẫn xấu

C. Excel button → Web modal
(em recommend)

  • Click button trong Excel
  • VBA open browser tới localhost:8100/quote/...
  • Browser render rich modal — đẹp như HTML mockups anh đã xem
  • Latency <1s (FastAPI ready)
  • Charts đẹp (Chart.js / Apex)
  • Reuse 62 endpoints email_engine sẵn có
✅ Best UX + dễ maintain

🛠️ Approach C flow (em recommend)

1. Anh click button "⚡ So giá" trong Excel row
↓
2. VBA: Shell "start http://localhost:8100/quote/compare?row=7"
↓
3. Browser opens (Chrome/Edge tab mới)
↓
4. FastAPI /quote/compare endpoint:
• Read row data từ Excel (cached / passed in URL)
• Query parquet via DuckDB (đã có FreightDB)
• Render Chart.js + table
↓
5. Anh xem modal đẹp, fast (<1s), close tab khi xong
→ Quay lại Excel work tiếp
Why approach C reuse infra sẵn có: email_engine đã có FastAPI 62 endpoints + DuckDB engine + parquet cleaned + Chart capability. Em chỉ ADD 4 endpoint mới cho 4 buttons. KHÔNG xây từ zero.

🎯 Em recommend

Build approach C — Excel button → Web modal. Lý do:

  • Visual: Web modal đẹp hơn UserForm 10x. Anh đã thấy HTML mockups rồi → web modal sẽ đẹp y vậy.
  • Speed: <1 giây vs 5-30s VBA Shell call.
  • Reuse: email_engine FastAPI + DuckDB + parquet đã sẵn sàng.
  • Maintain: Update endpoint Python easier hơn VBA.
  • Future-proof: Khi anh build webapp Next.js — endpoints reusable.
⏱️ Effort: ~3-4h M2.7 (4 endpoints + browser-launch VBA macros)

Vs Approach A (~6h xây từ zero, kết quả ugly + chậm).
